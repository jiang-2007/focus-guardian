## 📖 项目概述

### 问题背景

数字化办公时代，我们每天在电脑前度过 8+ 小时。但有多少时间真正用于工作？多少次被通知弹窗、短视频、游戏打断？数据不会骗人——但前提是你要有数据。

市面上的时间追踪工具，要么依赖云端同步（隐私风险），要么过于复杂（学习成本高），要么无法主动拦截干扰。

### 解决方案

**Focus Guardian（专注卫士）** 是一款开箱即用的 Windows 桌面工具：

- **无感记录** — 后台自动检测活跃窗口，用户零操作
- **主动拦截** — 专注模式下阻止娱乐软件干扰，而非仅仅统计
- **隐私优先** — 数据全量存于本地 SQLite，不联网、不上传
- **可视化报告** — 日/周/月多维度图表，清晰展示时间去向

### 技术理念

> "你无法管理你不测量的东西" — Peter Drucker
>
> Focus Guardian 遵循「测量 → 认知 → 改进」闭环：先让数据可见，再提供管控工具，最终帮助用户形成自律习惯。

### 差异化亮点

1. **记录 + 拦截一体化**
   同类产品要么只记录（RescueTime/ManicTime），要么只拦截（Cold Turkey/Forest）。Focus Guardian 将两者融合：先通过统计发现问题，再通过专注模式实施干预。

2. **番茄钟 x 智能名单**
   将 Pomodoro 工作法与黑白名单联动——专注阶段自动拦截娱乐软件，休息阶段自动解除，无需手动切换。

3. **零网络依赖**
   所有功能离线可用。数据库用 SQLite，图表用 tkinter Canvas 自绘。不会因为公司防火墙 / 网络不稳定而影响使用。

4. **面向普通用户的产品设计**
   5 步新手引导 + 预设模板一键导入 + 可视化管理界面，降低非技术用户的使用门槛。

## ✨ 核心功能

### 1. 智能活动监控

- 每秒检测当前活跃窗口，自动记录进程名 + 窗口标题
- 支持 Chrome / Edge / Firefox 浏览器识别
- 鼠标键盘空闲 5 分钟自动暂停计时
- 短于 3 秒的快速切换自动过滤，减少数据噪音

### 2. 分类与名单管理

- 预设 **70+ 常见软件模板**（40 款游戏 + 17 款娱乐软件 + 12 款工作软件）
- 一键从当前运行进程中添加，免去手动输入进程名
- 工作 / 中性 / 娱乐三分类，自动标注

### 3. 番茄钟专注模式

| 类型 | 专注 | 休息 | 场景 |
|------|:--:|:--:|------|
| 番茄钟 | 25min | 5min | 日常任务 |
| 长番茄钟 | 50min | 10min | 编程/写作 |
| 深度工作 | 90min | 20min | 复杂项目 |
| 强力小时 | 60min | 10min | 高效冲刺 |
| 短冲刺 | 15min | 3min | 碎片利用 |
| 自定义 | 任意 | 任意 | 自由掌控 |

- 专注阶段检测黑名单进程，弹出提醒或强制终止（严格模式）
- 4 轮后自动进入长休息（15 分钟）
- 休息结束自动进入下一轮，形成持续专注流

### 4. 多维数据看板

- **今日概览卡片**：总时长 / 使用程序数 / 专注次数 / 连续天数
- **本周柱状图**：每天使用时长趋势（Canvas 自绘）
- **分类饼图**：工作/中性/娱乐占比（甜甜圈图）
- **Top 5 排名**：今日最常使用程序

### 5. 日/周/月报表

- 按日、周、月切换视图
- 每条记录显示时间段 + 程序名 + 窗口标题 + 时长
- 支持删除单条、删除当天、导出 TXT / CSV

### 6. 每日限额 & 提醒

- 为任意程序设置每日使用时长上限
- 达到 80% 温和提醒，达到 100% 强制提醒
- 每 10 分钟再次提醒，数据次日 0 点自动重置

### 7. 系统托盘 & 开机自启

- 关闭窗口 = 缩小到通知区域，不占用任务栏
- 托盘图标颜色切换（绿=监控中，橙=专注中）
- 右键菜单快速操作，左键双击恢复窗口
- 注册表级别开机自启，设置页一键开关

## 📦 运行与部署

### 环境要求

- Windows 10 / 11（64 位）
- Python 3.10 及以上

### 快速启动

```bash
# 1. 克隆/下载项目
cd focus_guardian

# 2. 安装依赖（仅首次）
pip install -r requirements.txt

# 3. 启动
python src/main.py
# 或双击 run.bat
```

### 打包为 EXE

```bash
pip install pyinstaller
pyinstaller --onefile --noconsole --name "FocusGuardian" src/main.py
# EXE 在 dist/FocusGuardian.exe
```

详细打包与部署指南见 [docs/packaging_guide.md](docs/packaging_guide.md)

### 开机自启

设置页 → 勾选「开机自动启动」即可，或参照打包指南中的手动配置方法。

## 🧠 产品设计思路

### 设计原则

1. **渐进式介入** — 默认只统计不打扰，用户主动开启专注模式后才执行拦截，避免反感
2. **数据所有权** — 用户数据 100% 归用户所有，存储在可访问的 SQLite 文件中，随时可删除/导出
3. **降低认知负荷** — 预设模板覆盖 90% 常见软件，新手引导 5 步完成，无需阅读文档即可上手

### 技术选型考量

- **tkinter** 而非 Electron — 一个 50MB 的安装包 vs 200MB，启动快、内存低
- **SQLite** 而非 MySQL/PostgreSQL — 零配置，单文件备份，无需数据库服务
- **Canvas 自绘图表** 而非 matplotlib — 省去 100MB+ 的科学计算依赖，启动速度提升 3 倍
- **pystray + Pillow** 而非 Qt — 轻量级系统托盘，无需引入完整 GUI 框架

### 为什么纯本地

- 隐私是基本权利，不是可选功能
- 办公环境常有防火墙限制，本地工具不受网络影响
- 数据在用户手中，即使项目停止维护，数据依然可用
