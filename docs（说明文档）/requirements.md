# 系统使用统计 & 专注管控工具 — 项目需求文档

> 文档版本：v1.0  
> 创建日期：2026-05-30  
> 项目代号：Focus Guardian（专注卫士）

---

## 一、项目概述

### 1.1 项目目标

开发一款运行在 Windows 上的桌面工具，帮助用户：
- **了解自己**：自动统计电脑使用情况（打开了哪些程序/网页、用了多久）
- **管住自己**：设置黑白名单、开启专注模式阻止娱乐软件干扰
- **改进自己**：通过使用报表回顾时间花在哪里

### 1.2 核心价值

| 痛点 | 解决方案 |
|------|----------|
| 不知道时间花在哪了 | 自动后台记录，无需手动操作 |
| 想专注却总被打断 | 专注模式强制拦截娱乐软件 |
| 想约束自己但靠不住意志力 | 黑白名单 + 超时提醒双重约束 |
| 数据不想上传到网上 | 纯本地运行，所有数据存自己电脑 |

### 1.3 适用人群

- 学生：控制游戏/视频时间，专注学习
- 上班族：统计工作时间，减少摸鱼
- 自由职业者：了解时间分配，提高效率

## 二、功能需求（详细说明）

### 2.1 活动监控（核心模块）

**2.1.1 程序监控**
- 自动检测当前活跃窗口对应的程序名称（如 `chrome.exe`、`Code.exe`）
- 记录每个程序的**开启时间**和**关闭时间**（精确到秒）
- 自动识别程序切换：用户切走时结束上一条记录，开始新记录
- 忽略短于 3 秒的切换（防止快速切换产生大量碎片记录）

**2.1.2 网页监控**
- 识别当前活跃浏览器窗口的网页标题（如 "YouTube - Google Chrome"）
- 从浏览器标题中提取网站名称
- 同样记录开启时间、关闭时间
- 支持的浏览器：Chrome、Edge、Firefox

**2.1.3 空闲检测**
- 鼠标/键盘超过 N 分钟（默认 5 分钟）无操作时暂停计时
- 恢复操作后自动继续计时
- 空闲期间不计入任何程序的使用时长

### 2.2 分类与名单管理

**2.2.1 软件分类**
- 预设分类：
  - 🟢 **工作类**：IDE、Office、设计工具、笔记软件等
  - 🟡 **中性类**：文件管理器、终端、系统设置等（默认分类）
  - 🔴 **娱乐类**：游戏、视频、社交媒体、购物等

**2.2.2 黑名单**
- 用户可手动添加/删除程序到黑名单
- 预设常见娱乐软件的黑名单模板
- 黑名单中的程序在专注模式下会被拦截

**2.2.3 白名单**
- 用户可手动添加/删除程序到白名单
- 白名单中的程序在专注模式下不受影响
- 设置白名单后，专注模式切换到「仅允许白名单」模式

**2.2.4 自定义规则**
- 可为每个程序单独设置每日使用时长上限
- 可为特定网站设置限制

### 2.3 专注模式（核心功能）

**2.3.1 基本流程**
1. 用户设置专注时长（预设：25分钟 / 45分钟 / 60分钟 / 自定义）
2. 点击「开始专注」进入专注模式
3. 专注期间：
   - 黑名单软件被阻止启动
   - 正在运行的黑名单软件弹出提醒窗口
   - 尝试打开黑名单网站时弹出提醒
4. 专注结束后弹出完成提示，展示本次专注统计

**2.3.2 拦截行为**
- **软件拦截**：
  - 检测到黑名单进程时，弹出提醒窗口（非强制性）
  - 提醒窗口显示：「你正在专注模式中，确定要打开 [软件名] 吗？」
  - 用户可选择「继续专注」或「结束专注」
- **网站拦截**：
  - 检测到黑名单网站时，弹出类似提醒
  - 不强制关闭网页（避免数据丢失），但持续提醒

**2.3.3 专注统计**
- 记录每次专注的：开始时间、计划时长、实际时长、中断次数
- 专注历史可查看

**2.3.4 严格模式（可选）**
- 开启后真正阻止黑名单程序启动（强制关闭进程）
- 默认关闭，用户可在设置中开启

### 2.4 使用报表

**2.4.1 日报**
- 显示今日使用总时长
- 按程序/网站分类的饼图
- 每小时使用热力图
- 今日时间线（什么时间在用什么软件）

**2.4.2 历史记录**
- 按日期查看历史使用情况
- 支持日期范围筛选
- 列表视图展示每条使用记录（程序名、开始时间、结束时间、持续时长）

**2.4.3 数据管理**
- 删除指定日期的记录
- 删除指定程序的记录
- 清空所有历史数据（需二次确认）
- 设置数据保留天数（默认 90 天，自动清理旧数据）

### 2.5 每日限额

- 可为每个程序或分类设置每日使用时长上限
- 达到上限的 80% 时弹出温和提醒
- 达到上限的 100% 时弹出强制提醒
- 超限后仍可使用，但每隔 10 分钟再次提醒
- 每日 0 点自动重置

### 2.6 数据导出

- 支持导出今日使用记录为 **TXT 文本文件**
- 导出格式：时间 | 程序名称 | 时长
- 一键导出，保存到用户指定位置
- 可选导出 CSV 格式（方便 Excel 打开）

### 2.7 后台运行与自启

**2.7.1 后台运行**
- 关闭主窗口后自动缩小到系统托盘（通知区域）
- 托盘图标显示当前状态（监控中 / 专注中）
- 右键托盘图标可快速操作（开始专注 / 停止专注 / 打开面板 / 退出）

**2.7.2 开机自启**
- 可在设置中开启/关闭开机自启
- 自启后自动隐藏到系统托盘（静默启动）
- 通过 Windows 注册表 Run 键实现

## 三、非功能性需求

### 3.1 技术约束

| 约束 | 说明 |
|------|------|
| 运行环境 | Windows 10 / Windows 11 |
| 开发语言 | Python 3.10+ |
| 离线运行 | 全程不联网，不调用任何 API，不使用外部模型 |
| 资源占用 | 内存占用 < 100MB，CPU 占用 < 5%（后台运行时） |
| 存储 | 使用 SQLite 本地数据库 |

### 3.2 UI 设计要求

| 要求 | 说明 |
|------|------|
| 主题 | 浅色主题，背景白色/浅灰 |
| 主色调 | 淡绿色（#4CAF50 附近色系） |
| 字体 | 微软雅黑，清晰易读 |
| 布局 | 简洁直观，功能分区明确 |
| 操作 | 大部分操作用按钮/开关完成，减少学习成本 |
| 托盘 | 系统托盘图标绿色调，与主题一致 |

### 3.3 代码质量要求

- 面向代码新手友好：关键步骤有注释说明
- 函数单一职责，便于理解和修改
- 配置文件集中管理（不硬编码）
- 错误处理完善，异常情况有提示

## 四、数据模型概要

### 4.1 核心数据表

```
activity_log（活动记录表）
├── id            INTEGER PRIMARY KEY
├── process_name  TEXT      (进程名，如 chrome.exe)
├── window_title  TEXT      (窗口标题，如 "YouTube - Google Chrome")
├── category      TEXT      (分类：work / neutral / entertainment)
├── start_time    DATETIME  (开始时间)
├── end_time      DATETIME  (结束时间)
├── duration      INTEGER   (持续秒数)
└── date          TEXT      (日期，如 2026-05-30)

blacklist（黑名单）
├── id            INTEGER PRIMARY KEY
├── process_name  TEXT      (进程名)
├── display_name  TEXT      (显示名称)
└── category      TEXT      (分类)

whitelist（白名单）
├── id            INTEGER PRIMARY KEY
├── process_name  TEXT
└── display_name  TEXT

focus_sessions（专注记录）
├── id            INTEGER PRIMARY KEY
├── start_time    DATETIME
├── planned_duration INTEGER  (计划专注秒数)
├── actual_duration  INTEGER  (实际专注秒数)
├── interruptions INTEGER   (中断次数)
└── completed     BOOLEAN   (是否完成)

daily_limits（每日限额）
├── id            INTEGER PRIMARY KEY
├── process_name  TEXT
├── limit_minutes INTEGER
└── enabled       BOOLEAN

settings（设置表，键值对）
├── key           TEXT PRIMARY KEY
└── value         TEXT
```

## 六、术语说明

| 术语 | 说明 |
|------|------|
| 黑名单 | 专注模式下会被拦截的软件列表 |
| 白名单 | 专注模式下不受影响的软件列表 |
| 专注模式 | 计时器驱动的防干扰模式 |
| 严格模式 | 强制关闭黑名单进程（而非仅提醒） |
| 活跃窗口 | 当前用户正在操作的程序窗口 |
