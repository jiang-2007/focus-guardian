## Phase 1：基础框架 + 活动监控 + 数据存储

**目标**：让程序能在后台跑起来，检测当前活跃窗口并记录到数据库。

### 要创建/修改的文件
| 文件 | 作用 |
|------|------|
| `requirements.txt` | Python 依赖清单 |
| `src/config.py` | 全局配置（数据库路径、检测间隔等） |
| `src/__init__.py` | 包初始化 |
| `src/database/__init__.py` | 数据库包初始化 |
| `src/database/db_manager.py` | 数据库管理类 |
| `src/monitor/__init__.py` | 监控包初始化 |
| `src/monitor/activity_monitor.py` | 活动监控类 |
| `src/main.py` | 入口文件（先命令行测试） |

### 具体任务
1. ✅ 创建 `requirements.txt`，列出依赖包
2. ✅ 创建 `config.py`，定义所有配置项
3. ✅ 创建 `db_manager.py`，实现：
   - 初始化数据库（自动建表）
   - 插入活动记录
   - 查询今日记录
   - 关闭旧记录（设置 end_time）
4. ✅ 创建 `activity_monitor.py`，实现：
   - 每 1 秒检测当前活跃窗口
   - 获取进程名 + 窗口标题
   - 识别窗口切换，写入数据库
   - 忽略短于 3 秒的切换
5. ✅ 创建 `main.py` 命令行测试入口

### 验证标准
- 运行 `python main.py` 后，数据库中出现活动记录
- 切换窗口后能正确记录切换

## 开发注意事项

1. **每个 Phase 完成后自己测试一遍**，确认没问题再进入下一阶段
2. **每个 Phase 的代码可以独立运行**，不必等全部完成
3. **注释用中文写清楚**：每个函数干什么、参数什么意思
4. **配置文件集中管理**：修改配置只需改 `config.py`
5. **错误要友好提示**：用 `try/except` 捕获异常，弹出提示框而不是直接崩溃
